# Rapport — [TD2]

## Points clés

### 1. [Ce que le CSS doit ou ne doit pas et mettre ces bases]

J'ai compris que le css n'est pas sensé arranger une page html mais simplement l'améliorer dans son esthétique de présentation. Il vaut donc commencer par un css minimal, comme la typographie puis utiliser des balises tel que Flex ou Grid.

### 2. [Utiliser un layout simple et logique]

J'ai compris sur quand il faudrait que j'utilise Grid ou Flex selon la sturcture souhaité sur le site. Tout cela, sans casser l'odre du html. Le layout doit rester "réversible", si on enlève le css, la page doit rester lisible.
